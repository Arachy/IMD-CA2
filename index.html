Projects

During my time in college, I've worked on a number of projects, both in and outside of college, some of which I'll talk about here.

Food Delivery App

This project was the Second Year Project I completed as part of college. I worked as part of a team with 2 other classmates. We created a web application usign the Django framework in Python. The app was aimed at deli style restaurants/shops (Dunnes Stores, Centra, etc.). The idea of the app was to have restaurant managers sign up and add their restaurant and menu items to the database so that customers could order food from that restaurant through our website. Some of the features included having user accounts with different permissions so that customers couldn't access areas of the website that were meant only for restaurant managers.
Technologies & Tools Used: Python, Django, Crispy Forms, Stripe Payment API, Photoshop

App Opener

This was a solo app I created outside of college to improve workflow using the Tkinter framework. The app opens other apps that the user adds to a list. So instead of opening 5 or 6 different apps manually for work or college, this app does all that for you at the push of a button. Users could create profiles within the app so that you could have different pre-set lists of apps that you could load and open depending on who is using the computer or what they are doing.
Technologies & Tools Used: Python, Tkinter framework, OS framework, CSV framework

InsuranceSuite Transistion Explorer

This project was undertaken during my internship in Guidewire with the other interns there at the time. The idea was to allow developers within Guidewire and it's partners easily compare different versions of the InsuranceSuite products, which consist of PolicyCenter, BillingCenter, and ClaimCenter. As one of the first major projects of my internship, there was a lot of learning involved. This included learning the rough architecture of the InsuranceSuite products, learning React from scratch as I had never used it in college before, and learning internal tools such as Stash for source control and Jira for issue management, all while still making worthwhile contributions to the team and the app. The app allowed users to select a specific product (PolicyCenter, BillingCenter, or ClaimCenter) and then select a source and target version (consisting of both major and minor versions). These 2 product versions were then downloaded to our backend container and an interactive diff view was generated of them using a diff view NPM package.
Technologies & Tools Used: JavaScript, NodeJS, ReactJS, Docker, react-diff-view (https://www.npmjs.com/package/react-diff-view), some other internal tools
